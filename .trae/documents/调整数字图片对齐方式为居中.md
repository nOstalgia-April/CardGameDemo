用户指出数字图片的位置不对，需要对齐到 200x200 格子的中心（100x100）。

为了解决这个问题，我将修改 `scripts/level_select_cell.gd` 中 `level_number_rect` 的布局设置：

1.  **修改布局模式**：将 `level_number_rect` 的锚点预设（Anchors Preset）从 `PRESET_FULL_RECT` 改为 `PRESET_CENTER`。这将使控件的基准点位于父节点的正中心（即 100, 100）。
2.  **设置延伸方向**：设置 `grow_horizontal` 和 `grow_vertical` 为 `GROW_DIRECTION_BOTH`。这确保图片加载后，会以中心点为基准向四周延伸，从而保证图片的几何中心准确对齐到格子的中心。
3.  **移除冗余设置**：移除不再需要的 `expand_mode` 和 `stretch_mode` 设置，让 `TextureRect` 自然适应图片大小并居中。

**代码变更预览**：
```gdscript
# 修改前
level_number_rect.expand_mode = TextureRect.EXPAND_KEEP_SIZE
level_number_rect.stretch_mode = TextureRect.STRETCH_KEEP_CENTERED
level_number_rect.layout_mode = 1 # Anchors
level_number_rect.set_anchors_preset(Control.PRESET_FULL_RECT)

# 修改后
level_number_rect.layout_mode = 1 # Anchors
level_number_rect.anchors_preset = Control.PRESET_CENTER
level_number_rect.grow_horizontal = Control.GROW_DIRECTION_BOTH
level_number_rect.grow_vertical = Control.GROW_DIRECTION_BOTH
```
这将确保数字图片准确居中显示。
